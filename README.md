# TinyFramework

这是一个集成框架（就是拉几个库自己缝合一下），借鉴了很多BDFramework的思想，使用了依赖注入与事件分发的方法来达到游戏开发中单例太多，耦合度太高的问题。


至于为什么使用VContainer，不使用Zenject的原因是因为他更新，更快，表现良好~

注意！！
没有热更功能！！！主要为单机游戏提供良好可用的解决方案。
